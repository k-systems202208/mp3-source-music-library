# 変更履歴

## Phase 0: iTunes XML検索アプリ

### 出発点

- iTunesからエクスポートしたXMLを検索したい
- WebサーバーやDBを含め、可能な限り無料にしたい
- 対象は8,383曲

### 初期構成

- XMLをJSONへ変換
- JSONを単一HTMLへ埋め込み
- ブラウザJavaScriptだけで検索
- ローカルではダブルクリック
- サーバー・DB・ネット接続不要

### 初期機能

- 曲名・アーティスト・アルバム・作曲者検索
- ジャンル絞り込み
- 複数項目の並べ替え
- 件数・合計再生時間
- 追加読み込み
- 図書館のカード目録デザイン

## Phase 0.1: 曲カード整理

- 左側の先頭文字表示を削除
- 表示順を曲名→アルバム→アーティストへ
- 年・ジャンル等の常時表示を整理

## Phase 0.2: 関連リンクとフィルターチップ

- アルバム名・アーティスト名をリンク化
- 同じ対象へ即時絞り込み
- 上部に条件チップ
- 絞り込み範囲内で追加検索

## Phase 0.3: 表記確認・補正

- ローマ字曲名の自動日本語変換は誤変換リスクから不採用
- 曲名のインライン編集
- Google検索による正式表記確認
- 英数字タイトルのみ
- 訂正済みのみ
- localStorageへ補正保存
- 訂正済みラベル

## Phase 0.4: アーティスト補正

- アーティスト名の編集
- 同じ元アーティスト名の全曲へ一括反映
- localStorageへ保存
- 絞り込み表示名にも追従

## Phase 0.5: 3ビュー・ドリルダウン

- プルダウンを廃止
- 曲名・アーティスト・アルバムの3ボタン
- アーティスト→アルバム→曲
- アルバム→曲
- パンくず
- 曲カードから直接ドリルダウン

## Phase 1: 外部JSON版

- 埋め込みJSONを`music-library-data.json`へ分離
- `fetch()`で読込
- ローカルHTTPサーバー起動BAT
- HTMLを軽量化

## Phase 2: MP3プレーヤー版

- MP3再生ボタン
- 下部audioプレーヤー
- MP3インデックス生成

## Phase 2.1: 再帰走査・長いパス

- PowerShell走査からPythonへ変更
- `os.walk`
- 日本語・長いWindowsパス
- MP3・画像の再帰インデックス

## Phase 2.2: アートワーク

- 曲カード
- 下部プレーヤー
- アルバム代表画像
- embedded／folder／cover等

## Phase 2.3: JSONとMP3の照合問題

発見した問題:

- 相対パス正規化時にフォルダ情報を失う
- 重複ファイル名を判別できない
- 曲名`1/2`がパスとして扱われる
- JSON曲数とMP3数が一致しない

## Phase 3: MP3正本版

- 物理MP3ファイル1つを1曲
- ID3・ファイル名・フォルダから生成
- 旧JSONは補助情報
- 文字化け補正
- 診断CSV／JSON
- 埋め込み画像抽出

## Phase 3.1: 文字化け・再生修正

- Mutagen同梱
- CP932／UTF-8／UTF-16／Latin-1補正
- 曲IDを文字列へ統一
- MP3 Byte Range
- キャッシュ再生成

## Phase 4: SQLite v1

- `library.db`を管理情報の正本へ
- 再生回数・補正を永続化
- localStorage補正をサーバー共有へ発展
- 移動・改名検出
- 論理削除
- 日次バックアップ
- 互換JSON出力

## Phase 5: SQLite API v2

- 全曲一括JSON読込を廃止
- `/api/browse`
- 80件ページ取得
- 検索・集計・並べ替えをSQLiteへ
- 起動時全曲JSON生成を廃止

## v2.1

- 表示順からトラック番号順を削除
- アルバム内は自動曲順
- 最大化プレーヤー
- モバイル全画面
