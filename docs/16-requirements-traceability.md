## 3. UI要件の変遷

### フィルターチップからドリルダウンへ

初期:

```text
曲カードのアーティストをクリック
→ 「アーティスト: ○○ ×」のチップ
→ 同じ画面内で絞り込み
```

現行:

```text
曲カードのアーティストをクリック
→ アーティストのアルバム一覧
→ アルバムの曲一覧
→ パンくずで戻る
```

### localStorageからSQLiteへ

初期:

```text
端末Aで補正
→ 端末Aだけに保存
```

現行:

```text
端末Aで補正
→ SQLiteへ保存
→ PC・端末Bにも反映
```

### 全件データからAPIページングへ

初期:

```text
8,383曲をブラウザへ全部読み込む
→ 描画だけ分割
```

現行:

```text
検索条件をSQLiteへ送る
→ 80件だけ受信
→ スクロール時に次の80件
```

## 4. 未完了・再検討候補

初期版にあり、現行UIでは専用機能として残していないもの:

- ジャンル選択による絞り込み
- 年順
- 再生時間順
- 年・ジャンルの常時カード表示

これらは削除漏れではなく、UI簡素化と利用頻度を考慮して整理したものです。

再追加する場合は、SQLite APIの検索条件と索引件数集計へ組み込みます。

## 5. 要件判断の原則

1. 元データを破壊しない
2. 誤認識する自動化より確認可能な補正
3. 大量データを全件転送しない
4. 同じ概念を複数の正本で管理しない
5. 接続できる範囲をネットワークで限定する
6. 機能追加時は既存の検索・再生を壊さない
