# アプリ仕様書

## 1. 文書情報

| 項目 | 内容 |
|---|---|
| システム名 | MP3 Source Music Library |
| 対象版 | SQLite API v2.4 |
| 対応DBスキーマ | version 4 |
| 主対象OS | Windows |
| クライアント | モダンブラウザ |
| 音源形式 | MP3 |
| 想定規模 | 数千～数万曲の個人ライブラリ |

## 2. 目的

PC上に保存したMP3を、専用音楽管理ソフトへ取り込むことなく、ブラウザから高速に検索・再生できるようにします。


## 2.1 初期要件との関係

初期版はiTunes XML由来データの検索専用アプリでした。

初期実装に含まれていた要件:

- 8,383曲のリアルタイム検索
- 曲名・アーティスト・アルバム・作曲者
- ジャンル絞り込み
- 年・追加日・再生回数・長さ等の並べ替え
- 件数・合計再生時間
- 追加読み込み
- カード目録デザイン
- アルバム・アーティストのリンク絞り込み
- localStorageによる曲名・アーティスト補正
- Google検索による正式表記確認
- 英数字タイトル・訂正済みの絞り込み
- 曲・アーティスト・アルバムの3ビュー
- パンくず付きドリルダウン
- 外部JSON読込

現行版では、検索処理と補正保存をSQLiteへ移しました。ジャンル専用フィルター、年順、長さ順は現行UIでは提供していませんが、genre・year・durationはDBへ保持しています。

詳細な対応は[要件トレーサビリティ](16-requirements-traceability.md)を参照してください。

## 4. 機能一覧

### 4.1 ライブラリ生成

|---|---|---|

| F-003 | タグ解析 | MutagenでID3v2.2／v2.3／v2.4を解析 |
| F-004 | フォールバック解析 | Mutagen失敗時に簡易ID3v2、ID3v1を使用 |
| F-005 | メタデータ補完 | タグ不足時にファイル名・フォルダ名から補完 |
| F-006 | 文字化け補正 | UTF-8、CP932、Latin-1等の誤判定候補を評価 |
| F-007 | 差分スキャン | サイズ・更新日時が同じファイルは再解析しない |
| F-008 | 移動検出 | 内容署名が一意一致した場合、同一曲としてパス更新 |
| F-009 | 論理削除 | 未検出曲を`is_available=0`として履歴保持 |
| F-010 | 旧JSON移行 | 新規DB行に限り、再生回数・追加日等を高確度で継承 |

## 5. 画面仕様

### 5.1 表示タブ

- 曲
- アーティスト
- アルバム

アーティストを選ぶとアルバム一覧、アルバムを選ぶと曲一覧へ遷移します。

### 5.2 曲カード

- アートワーク
- 曲名
- アーティスト
- アルバム
- 再生時間等
- 再生ボタン
- 曲名・アーティスト補正ボタン

### 5.3 下部・最大化プレーヤー

- アートワーク
- 曲名
- アーティスト
- 前曲、次曲
- シャッフル
- 全体リピート
- 1曲リピート
- Audio標準コントロール
- 最大化／縮小

開閉しても同一audio要素を使用し、再生位置を維持します。

## 6. メタデータ優先順位

| 項目 | 優先順位 |
|---|---|
| 曲名 | TIT2 → ID3v1 → ファイル名推定 → stem |
| アーティスト | TPE1 → ID3v1 → `Artist - Title` → 不明 |
| アルバム | TALB → ID3v1 → 親フォルダ |
| トラック | TRCK → ファイル名先頭番号 |
| ディスク | TPOS → Disc/Disk/CDフォルダ |
| 再生時間 | Mutagen → MPEGフレーム推定 |

## 7. 非機能要件

### 性能

- 1回の取得件数は既定80、最大200
- 未変更MP3はタグを再読込しない
- ブラウザへ全曲を一括送信しない

### 信頼性

- SQLiteトランザクション
- 外部キー有効
- WALモード
- busy timeout 30秒
- 既存曲の解析失敗時は前回情報を維持
- 削除は論理削除

### セキュリティ

- 既定はlocalhostのみ
- DB、ソース、BAT、バックアップを静的公開しない
- 外部公開はTailscaleを推奨
- アプリ独自認証はない

## 8. 複数利用者仕様

- 同時に別曲を再生可能
- プレーヤー状態は端末別
- 再生回数・補正は共有
- 誰が再生したかは記録しない
- 同じ項目の同時補正は最後の更新が有効

## 9. 音質仕様

- MP3をそのままHTTP配信
- サーバーでデコード・エンコードしない
- 低速通信時は音質低下ではなくバッファリング
- Bluetoothコーデックによる再圧縮は端末側の事情

## 10. 制約・既知の限界

- 走査対象はMP3のみ
- アプリ内ログインなし
- HTTPSはTailscale Serve等に依存
- PCの電源・サーバープロセスが必要
- 全曲取得用`GET /api/tracks`は互換用
- `favorite`、`rating`、`album_override`はUI未実装
