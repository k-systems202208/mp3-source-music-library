# UI／UX設計の変遷

## 1. デザインテーマ

初期から一貫して、図書館のカード目録をモチーフにしています。

- 1曲を1枚のカードとして扱う
- 紙の目録に近い落ち着いた配色
- タイトルを最も強く表示
- メタデータは補助情報として整理
- 一覧から詳細へ段階的に進む

## 2. 初期カード

初期版では左側へ頭文字を大きく表示していました。

利用者から「先頭1文字は不要」と判断され、カード内の視線移動を減らすため削除しました。

情報順は次へ整理しました。

```text
曲名
アルバム
アーティスト
```

## 3. フィルターリンク

曲カード上のアルバム・アーティストをリンク化しました。

### 利点

- 新しい検索語を入力せず絞り込める
- 音楽ライブラリを関連情報から辿れる
- 現在条件をフィルターチップで確認できる

### 課題

- 曲・アーティスト・アルバムという異なる一覧構造が同じ画面に混在
- 深い階層を表現しにくい
- 戻る操作がフィルター解除に依存

この課題からドリルダウン式へ移行しました。

## 4. 3ビュー方式

プルダウンをなくし、主要操作を次の3ボタンへ集約しました。

- 曲名
- アーティスト
- アルバム

アーティストは「一覧→アルバム→曲」、アルバムは「一覧→曲」としました。

### パンくず

パンくずにより、現在の場所と戻り先を明確にしました。

```text
アーティスト一覧 › アーティスト名 › アルバム名
```

## 5. 表記補正UX

### 自動変換を採用しなかった理由

ローマ字表記が必ず誤りとは限りません。機械的な日本語変換は、正式な英語曲名を誤変換する可能性があります。

### 採用した流れ

```text
英数字タイトルのみで候補を絞る
→ Google検索で正式表記を確認
→ 編集ボタンで補正
→ 訂正済みラベル
→ 訂正済みだけを再確認
```

これは、速さだけでなく正確性と訂正可能性を重視した設計です。

## 6. アーティスト補正

表記ゆれは1曲ずつ直さず、アーティスト単位で一括補正します。

初期版では「同じ文字列」、現行版では「同じartist_id」を単位とします。

後者により、元タグを保持しながら正規化された同一アーティストへ安定して反映できます。

## 7. 索引の再導入

最初のカード左側の頭文字は不要として削除しましたが、大量ライブラリでは一覧を素早く移動する索引自体は有用でした。

そこで、カード装飾としてではなく、独立した操作部品として再設計しました。

```text
0-9 A-Z
あ か さ た な は ま や ら わ
漢字・他
```

該当件数0の索引は無効化し、現在の検索範囲と連動します。

## 8. プレーヤーUI

### 下部プレーヤー

検索結果を見ながら再生を継続するため、画面下部へ固定しました。

### 最大化

アートワークや曲情報を大きく見たい場面に対応し、同じaudio要素をモーダルへ見せる構成にしました。

モーダル開閉時に再生位置を失いません。

### 再生モード

- 前曲
- 次曲
- シャッフル
- 全体リピート
- 1曲リピート

操作状態は端末ごとに保存し、ほかの利用者の再生へ影響させません。

## 9. モバイル

- タッチしやすいボタン
- 索引は横スクロール
- 最大化プレーヤーは全画面
- 下部プレーヤーを維持
- OSのMedia Sessionへ連携

音源と履歴は共有しますが、再生状態は端末別です。

## 10. 現在のUI原則

- 主要操作を画面上部へ集約
- 検索結果を止めずに再生
- 階層はパンくずで示す
- 補正は元値を残す
- 読みがないデータを無理に推測しない
- 件数0の操作を無効化する
- PCとスマホで同じ概念を保つ
