MP3正本版 音楽ライブラリ SQLite API v2.4
============================================

【この版の位置づけ】
Musicフォルダ内のMP3を物理ファイルの正本、library.dbを管理情報の正本として扱います。

v2では、ブラウザへ全曲を一括送信しません。検索・絞り込み・表示順・ページ取得を
SQLite側で処理し、画面には80件ずつ返します。8,000曲以上あっても、起動直後に
全曲JSON・MP3パス一覧・画像パス一覧を読み込む必要がありません。


【SQLite v1 / v2からの更新方法】
1. 音楽ライブラリの黒い画面とブラウザを閉じます。
2. 念のため backup-library.bat を実行します。
3. このZIPを展開し、中の mp3_source_player_sqlite フォルダのファイルを、
   現在使用中の同名フォルダへ上書きコピーします。
4. Music、library.db、.artwork-cache、BackupsはZIPに含まれていないため、
   上書きしても既存データは残ります。
5. start-music-library.bat を実行します。

別の場所へ新規展開する場合は、旧フォルダから次をコピーしてください。
- Music
- library.db（再生回数・補正内容を引き継ぐ場合）
- .artwork-cache（省略可能。再生成されます）
- Backups（過去バックアップも移す場合）


【通常の使い方】
1. start-music-library.bat をダブルクリックします。
2. MP3の追加・削除・変更をSQLiteへ反映します。
3. ブラウザが自動で開きます。
4. 利用中は黒い画面を閉じないでください。

変更されていないMP3はファイルサイズと更新日時を確認し、タグを読み直しません。


【v2でSQLite APIへ移した処理】
- 曲名・アーティスト・アルバム・作曲者の検索
- 英数字タイトルのみ
- 訂正済みのみ
- 曲名順
- アーティスト順
- アルバム順
- 再生回数順
- 追加日順
- アーティスト一覧の集計
- アーティスト別アルバム一覧の集計
- アルバム一覧の集計
- 80件単位のページ取得
- 検索結果全体の曲数と合計再生時間の集計

画面上部に「SQLite APIページ取得」と表示されればv2で動作しています。


【v2.1の画面変更】
- 「表示順」から「トラック番号順」を削除しました。
- アルバムを開いたときは、選択肢を表示せずアルバム内の曲順で自動表示します。
- 下部プレーヤーのアートワークまたは曲名部分をクリックすると、最大化モーダルを開きます。
- 下部プレーヤーの「⛶」ボタンからも最大化できます。
- モーダルを閉じても再生位置と再生状態は維持されます。
- 右上の×、背景部分のクリック、Escキーで元の大きさに戻ります。
- スマートフォンではモーダルを画面いっぱいに表示します。


【v2.2の頭文字索引】
- アーティスト一覧と全体のアルバム一覧に、アルファベット索引と五十音索引を追加しました。
- 0-9、A～Z、あ・か・さ・た・な・は・ま・や・ら・わ、漢字・他から絞り込めます。
- 該当項目がない文字は無効表示になります。
- 選択した文字は検索語と併用でき、「すべて」で解除できます。
- The / A / Anで始まる英語名は、次の語の頭文字で分類します。
  例: The Beatles は B、The Wall は Wです。
- ひらがなとカタカナは同じ五十音として扱い、濁音・半濁音も同じ行へまとめます。
- ID3のアーティスト並べ替えタグ（TSOP）とアルバム並べ替えタグ（TSOA）に対応しました。
  漢字名でも読みがタグに保存されていれば五十音へ分類されます。
- 読みタグのない漢字名は「漢字・他」に分類します。
- v2.2への初回更新時だけ、既存MP3の並べ替えタグを取得するため全曲のタグを再確認します。
  2回目以降は従来どおりSQLiteキャッシュが使われます。


【v2.3の曲名索引】
- 曲一覧にもアルファベット索引と五十音索引を追加しました。
- 全曲一覧だけでなく、検索結果、選択中のアーティスト、選択中のアルバムの範囲内で利用できます。
- 索引を選ぶと、現在の条件に該当する曲名だけをSQLite側で抽出します。
- ID3の曲名並べ替えタグ（TSOT）に対応しました。漢字の曲名でも読みが保存されていれば五十音へ分類されます。
- 画面で曲名を補正した場合は、補正後の曲名を優先して索引へ分類します。
- v2.3への初回更新時だけ、既存MP3のTSOTを取得するため全曲のタグを再確認します。
  2回目以降はSQLiteキャッシュが使われます。


【v2.4の再生モード】
- 下部プレーヤーと最大化プレーヤーに「前へ」「次へ」を追加しました。
- シャッフル再生は、再生を開始した時点の検索・索引・アーティスト・アルバム範囲を対象にします。
- シャッフル中は同じ曲を重複させずに一巡し、全体リピートがオンの場合だけ次の一巡へ進みます。
- 全体リピートは、現在の対象範囲の末尾から先頭へ戻ります。
- 1曲リピートは、曲が終了すると同じ曲を先頭から再生します。
- 全体リピートと1曲リピートは同時に有効にならず、選択中のボタンが金色になります。
- 再生モードはブラウザへ保存され、次回起動時にも維持されます。
- スマートフォン等でMedia Session APIに対応するブラウザでは、端末側の前曲・次曲操作にも対応します。
- 「前へ」は再生開始から5秒を超えている場合、まず現在の曲の先頭へ戻ります。


【正本と役割】
Musicフォルダ
  MP3の正本です。物理MP3ファイル1つを1曲として扱います。

library.db
  曲情報、ファイル状態、再生回数、補正内容、スキャン履歴を保存する正本です。

legacy-library-data.json
  初回登録時に旧再生回数・追加日などを引き継ぐ補助辞書です。
  MP3の存在判定や曲数の正本には使用しません。

v1で毎回生成していた次のファイルはv2では不要なため、起動時に削除します。
- music-library-data.json
- mp3-index.json
- artwork-index.json

JSONが必要な場合は export-library-json.bat を実行してください。
Exportsフォルダへ日時付きJSONを作成します。


【SQLiteに保存する主な情報】
- MP3の相対パス、ファイルサイズ、更新日時、内容署名
- 曲名、アーティスト、アルバム、トラック番号、ディスク番号
- 再生時間、ジャンル、作曲者、年
- アートワークの場所と種類
- タグ・ファイル名・フォルダ名のどこから情報を取得したか
- 再生回数、最終再生日時、追加日
- 画面で補正した曲名・アーティスト名
- ファイルが現在存在するか
- スキャン履歴と診断内容


【再生回数と表記補正】
新しい曲の再生を開始するとtracks.play_countを1増やします。
一時停止からの再開では増えません。

曲名の鉛筆ボタンはtracks.title_overrideへ保存します。
アーティスト名の鉛筆ボタンはartists.display_name_overrideへ保存し、
同じartist_idの全曲へ反映します。

ブラウザやポート番号が変わっても、同じlibrary.dbを使用すれば保持されます。


【MP3の移動・改名・削除】
ファイルサイズと先頭・末尾データから軽量な内容署名を作ります。
旧パスから消えた曲と新規パスの署名が一意に一致した場合は移動と判断し、
曲ID・再生回数・補正内容を維持します。

見つからなくなったMP3は即時削除せず、is_available=0として履歴を残します。
画面にはis_available=1の曲だけ表示します。


【アートワーク】
優先順位は次のとおりです。
1. MP3埋め込み画像
2. 同じフォルダまたは親フォルダのfolder / cover / front / albumart / small
3. その他の画像

埋め込み画像は.artwork-cacheへ抽出します。
画面はSQLiteが返した相対パスから直接画像URLを作るため、画像一覧JSONは不要です。


【自動生成されるもの】
- library.db                    SQLite正本
- library.db-wal / library.db-shm
                                起動中にSQLiteが使用する一時ファイル
- library-diagnostics.json     最新スキャンの集計と診断
- library-diagnostics.csv      Excel等で確認できる診断一覧
- .artwork-cache               MP3埋め込み画像の抽出先
- Backups                      SQLiteバックアップ先
- Exports                      手動JSONエクスポート先


【バックアップ】
起動時、既存のlibrary.dbについて1日1回バックアップを作成します。
保存先は Backups/library-YYYYMMDD.db です。

手動バックアップは backup-library.bat を実行します。
MusicフォルダのMP3本体はSQLiteバックアップに含まれません。


【診断・メンテナンス】
コマンドプロンプトで次を実行できます。

  py -3 library-maintenance.py stats
  曲数、利用可能曲数、総再生回数などを表示します。

  py -3 library-maintenance.py check
  SQLite整合性を検査します。okなら正常です。

  py -3 library-maintenance.py backup
  手動バックアップを作成します。

  py -3 library-maintenance.py vacuum
  DBを最適化します。サーバーを閉じてから実行してください。

  py -3 library-maintenance.py export
  全曲の互換JSONをExportsフォルダへ出力します。


【文字化け対策】
- 同梱MutagenでID3v2.2 / v2.3 / v2.4を解析します。
- UTF-8、UTF-16、CP932、Latin-1の誤判定を補正します。
- 文字化けが残り、旧JSONと高確度で一致した場合のみ旧表記を補助します。

問題が残るファイルはlibrary-diagnostics.csvを確認してください。


【安全性】
初期設定では127.0.0.1だけで待ち受け、このPC自身からのみ接続できます。
library.db、Pythonソース、BAT、Backups、Exportsはブラウザから直接取得できません。

スマホ・タブレット向けに同一Wi-Fiへ公開するときは、認証とWindowsファイアウォールを
追加してから設定してください。


【同梱ライブラリ】
vendor/mutagenはMP3タグ解析用です。
ライセンスはvendor/MUTAGEN_LICENSE.txtを参照してください。
別途pip installは不要です。
